# スキル: 実行計画の完了

> **実行タイミング**: 実行計画のすべての受け入れ条件を満たし、PR がマージされたとき
>
> **目的**: 完了した作業を履歴として記録し、CONTEXT.md の「現在フェーズ・優先タスク」を最新状態に保つ。
>
> **前提**:
> - 対象の実行計画が `exec-plans/active/` に存在すること
> - すべての受け入れ条件を満たしていること（`pre-pr` スキルでチェック済みであること）
> - PR がマージ済みであること

---

## このスキルがすること

1. 対象の実行計画を確認する
2. 受け入れ条件の充足を最終確認する
3. ファイルを `active/` から `completed/` へ移動する
4. `docs/06_ai_context/CONTEXT.md` を更新する
5. 次の作業を案内する

---

## 手順

### ステップ 1: 対象計画の確認

`exec-plans/active/` のファイルを一覧し、完了する計画を確認する。

- ファイルが1件のみの場合はそれを選択する
- 複数ある場合は選択を促す

### ステップ 2: 受け入れ条件の最終確認

対象の実行計画ファイルを読み込み、受け入れ条件の充足を確認する。

- すべての条件が `- [x]` になっているか確認する
- 未チェックの条件がある場合は、完了できない旨を伝えて対処を促す
  - 条件が不要と判断する場合は、判断ログにその理由を記録してから進める

### ステップ 3: ファイルの更新と移動

対象ファイルを以下の順で更新・移動する。

1. フロントマターを更新する:
   ```yaml
   ---
   status: completed
   created: YYYY-MM-DD
   completed: YYYY-MM-DD   ← 今日の日付を記入
   ---
   ```
2. 進捗ログに完了エントリを追記する:
   ```markdown
   ### YYYY-MM-DD
   - すべての受け入れ条件を満たした。計画を完了状態に移行した。
   ```
3. ファイルを `exec-plans/completed/` に移動する（`exec-plans/completed/` がなければ作成する）

### ステップ 4: CONTEXT.md の更新

`docs/06_ai_context/CONTEXT.md` の「現在フェーズ・優先タスク」を更新する。

- 完了した計画を優先タスクから削除する
- 残っている `exec-plans/active/` の計画を確認し、次の優先タスクを反映する
- `active/` が空になった場合は「優先タスク: 次の exec-plan を作成してください」と記載する
- フェーズが移行する場合（例: Phase 1 → Phase 2）は現在フェーズも更新する

### ステップ 5: 次の作業の案内

以下の情報をもとに次のアクションを案内する。

| 状況 | 案内 |
|------|------|
| `exec-plans/active/` に他の計画がある | 次の計画を `start-feature` で開始することを案内 |
| `exec-plans/active/` が空 | `create-exec-plan` で次の計画を作成することを案内 |
| フェーズ移行が発生した | 次フェーズで必要なドキュメントの作成を案内 |

---

## 完了条件

- [ ] 対象ファイルのフロントマターが `status: completed`・`completed: YYYY-MM-DD` になっている
- [ ] ファイルが `exec-plans/completed/` に移動されている
- [ ] `docs/06_ai_context/CONTEXT.md` の優先タスクが更新されている
- [ ] 次のアクションが案内されている

完了後にエージェントが出力する報告:

```
=== 実行計画を完了しました ===

完了した計画 : exec-plans/completed/YYYY-MM-{name}.md
CONTEXT.md  : 優先タスクを更新しました

次のアクション: {状況に応じた案内}
```
