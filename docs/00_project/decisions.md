## ADR-001: UIフレームワークに WPF を選択

- **日付**: 2026-04-12
- **ステータス**: 採用済み
- **決定**: Windows UIフレームワークとして WPF（Windows Presentation Foundation）を採用する

### 背景
.NET 8 Windows デスクトップアプリの選択肢として WPF / WinUI 3 / MAUI がある。

### 理由
- WPFは.NET 8で継続サポートされており、成熟したエコシステムを持つ
- オーバーレイウィンドウの実装（透明度・最前面固定・クリックスルー）において実績・情報が豊富
- WinUI 3はまだWPFに比べてAPIの安定性・サードパーティ対応が不十分
- MAUIはデスクトップ専用用途には適していない

### 却下した選択肢
- **WinUI 3**: APIの成熟度不足、MSIX依存が複雑
- **MAUI**: マルチプラットフォーム向けであり、Windowsネイティブ機能との連携が煩雑

## ADR-002: MVVMライブラリに CommunityToolkit.Mvvm を選択

- **日付**: 2026-04-12
- **ステータス**: 採用済み
- **決定**: MVVMパターンの実装に CommunityToolkit.Mvvm を使用する

### 背景
WPF + MVVM の実装ライブラリとして Prism / ReactiveUI / CommunityToolkit.Mvvm などがある。

### 理由
- Microsoftが公式にメンテナンスするオープンソースライブラリであり、.NET 8対応
- Source Generatorによりボイラープレートを最小化できる
- 学習コストが低く、AIがコード生成しやすい
- 依存関係がシンプル

### 却下した選択肢
- **Prism**: 機能過多、DI統合が複雑
- **ReactiveUI**: Rx（リアクティブ）の学習コストが高い

## ADR-003: データ永続化に SQLite を選択

- **日付**: 2026-04-12
- **ステータス**: 採用済み
- **決定**: 作業リスト・設定の永続化に SQLite（Microsoft.Data.Sqlite）を使用する

### 背景
ローカルデータ保存の選択肢として JSON/XMLファイル・SQLite・LiteDBなどがある。

### 理由
- 構造化データのクエリ・マイグレーションが容易
- .NET公式サポート（Microsoft.Data.Sqlite）があり信頼性が高い
- 将来的なデータ追加・変更に対してスキーマ管理が可能

### 却下した選択肢
- **JSONファイル**: スキーマなし、マイグレーション困難
- **LiteDB**: サードパーティ依存、長期サポート不明

## ADR-004: 配布形式に MSIX を選択

- **日付**: 2026-04-12
- **ステータス**: 採用済み
- **決定**: アプリの配布パッケージ形式として MSIX を採用する

### 理由
- Windows 10/11でサポートされるモダンなパッケージ形式
- クリーンインストール・アンインストールが保証される
- 将来的な Microsoft Store 配布への対応が容易

### 却下した選択肢
- **インストーラなし（zip展開）**: 自動アップデートが困難、UACの扱いが煩雑
- **Inno Setup（EXEインストーラ）**: レガシー形式、Storeへの道が閉じる
