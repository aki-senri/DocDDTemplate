# 用語定義

> status: active

AIがドキュメントや会話を誤解しないよう、プロジェクト固有の用語を定義します。

---

## 用語一覧

| 用語 | 定義 | 注意 |
|------|------|------|
| **作業（Task）** | ユーザーが現在取り組んでいる、または取り組む予定のひとまとまりの活動。自由テキストで表現される。 | タスク管理ツールの「チケット」とは異なり、粒度は問わない |
| **フォーカス表示** | 現在の作業名をPC画面上にオーバーレイで常時表示すること。他のウィンドウの上に重ねて表示される。 | 「常時表示」「オーバーレイ表示」と同義 |
| **オーバーレイウィンドウ** | WPFの最前面ウィンドウとして描画される、透明度や位置を自由に設定できる表示領域。 | タスクバーには表示されない |
| **作業リスト** | ユーザーが登録した再利用可能な作業の一覧。次回以降の作業入力で選択できる。 | 単なる「履歴」とは区別される（手動登録） |
| **履歴** | 過去に入力・選択した作業の自動記録。明示的に登録したリストとは異なる。 | |
| **設定ウィンドウ** | アプリの動作・表示をカスタマイズするための別ウィンドウ。オーバーレイとは独立して開く。 | 「設定画面」と同義 |
| **アクティブウィンドウ監視** | Windowsのフォアグラウンドウィンドウを監視し、許可外のアプリに切り替わったことを検出する機能。 | オプション機能（設定でON/OFF可能） |
| **注意アラート** | アクティブウィンドウ監視が許可外の切り替えを検出したとき、オーバーレイの表示スタイルを変えて視覚的に警告すること。 | 通知音・ポップアップは含まない（設定で追加可能） |
| **MSIX** | Windowsアプリの配布パッケージ形式。サンドボックス化・クリーンアンインストールに対応。 | |
| **CommunityToolkit.Mvvm** | MVVMパターンの実装を支援するMicrosoft公式のオープンソースライブラリ。 | 旧称 Microsoft.Toolkit.Mvvm |
| **ViewModel** | MVVMにおけるView（UI）とModel（ビジネスロジック）の橋渡し役。データバインディングのソースとなる。 | |
| **トースト通知** | Windows標準の通知バナー。本アプリでは使用しない（オーバーレイ内で完結させる）。 | |
