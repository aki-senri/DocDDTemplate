# constraints.md — 制約・非対応事項

## 技術制約

| ID     | 制約                                                                    |
|--------|-------------------------------------------------------------------------|
| TC-001 | 対象 OS は Windows 10 22H2 以降 / Windows 11 のみ。macOS・Linux は非対象。 |
| TC-002 | .NET 9 ランタイムが対象環境にインストールされていること。                   |
| TC-003 | NTFS または ReFS ファイルシステムでのみ動作を保証する。                    |
| TC-004 | フォルダ直下のファイル数上限は 10,000 件とする（パフォーマンス上の目安）。  |

## ビジネス制約

| ID     | 制約                                                                    |
|--------|-------------------------------------------------------------------------|
| BC-001 | タグデータは `.filetags` ファイルのみに保存し、外部サービスには送信しない。 |
| BC-002 | ファイルの削除・リネーム操作はスコープ外とする（移動のみ提供）。             |
| BC-003 | 管理対象は開いたフォルダ直下のファイルのみ。サブフォルダ以下は対象外。      |

## 非対応事項（非ゴール）

以下は意図的にサポートしない機能。将来の拡張で検討する余地はあるが、
現バージョンでは実装しない。

- **サブフォルダの再帰的な走査・管理**
- **ファイルのコピー・削除・リネーム**
- **クラウドストレージ（OneDrive / Google Drive 等）との連携**
- **複数フォルダの同時管理**
- **タグの階層構造（親子タグ）**
- **ファイルのプレビュー表示**
- **検索機能（ファイル名・内容の全文検索）**
- **ネットワークドライブ上のフォルダのサポート**（動作は未保証）

## パフォーマンス要件

| ID     | 要件                                                              |
|--------|-------------------------------------------------------------------|
| PF-001 | フォルダオープン時（1,000 ファイル）の一覧表示は 2 秒以内に完了する。 |
| PF-002 | タグの付与・削除操作は 100 ms 以内に UI に反映される。               |
| PF-003 | `.filetags` ファイルの保存は 500 ms 以内に完了する。                 |

## セキュリティ制約

| ID     | 制約                                                                  |
|--------|-----------------------------------------------------------------------|
| SC-001 | アプリは管理者権限なしで動作すること（UAC 昇格不要）。                  |
| SC-002 | `.filetags` ファイルの内容は平文 JSON とし、暗号化は行わない。          |
| SC-003 | 読み取り権限のないフォルダを開こうとした場合は、エラーメッセージを表示する。 |
