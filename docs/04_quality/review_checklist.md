# コードレビューチェックリスト

> status: active

PRレビュー時に確認する項目です。エージェントによる自動レビューでも使用します。

---

## 共通チェック

### 正確性
- [ ] 実装がユーザーストーリー・受け入れ条件を満たしているか
- [ ] エッジケース（null、空文字、最大値）が考慮されているか
- [ ] エラーハンドリングが適切か（サイレントに失敗していないか）

### アーキテクチャ・設計
- [ ] `invariants.md` の不変条件に違反していないか
- [ ] 適切なレイヤーにコードが配置されているか（View/ViewModel/Service/Repository）
- [ ] DI（依存性注入）が使用されているか（`new` によるインスタンス化をしていないか）

### コード品質
- [ ] `coding_standards.md` の命名規則に従っているか
- [ ] 1ファイルが300行以内か
- [ ] `async void` の不適切な使用がないか
- [ ] `CancellationToken` を受け取れるAPIで渡しているか

### テスト
- [ ] 新しいビジネスロジックに対応するユニットテストが追加されているか
- [ ] テストメソッドの命名規則 `{メソッド}_{シナリオ}_{結果}` に従っているか
- [ ] モックが適切に設定されているか（検証なしのモックは避ける）

---

## WPF / XAML チェック

- [ ] ViewModel に View への直接参照がないか
- [ ] コードビハインド（.xaml.cs）にロジックが書かれていないか
- [ ] バインディングに `Mode=TwoWay` が必要な箇所で設定されているか
- [ ] リソース（色・サイズ）が直接記述されず、ResourceDictionary を参照しているか
- [ ] `INotifyPropertyChanged` が正しく発火しているか（CommunityToolkit.Mvvm の `[ObservableProperty]` 使用）

---

## SQLite / データアクセスチェック

- [ ] DB書き込みがトランザクション内で行われているか
- [ ] `using` / `await using` でコネクションが確実にDispose されているか
- [ ] SQLインジェクション対策（パラメータバインディング）がされているか
- [ ] マイグレーション（DDL変更）が `DatabaseInitializer` に反映されているか

---

## セキュリティチェック

- [ ] 外部ネットワーク通信が発生していないか
- [ ] ユーザー入力が Service 層でバリデーションされているか
- [ ] `%LOCALAPPDATA%` 以外の場所にファイルを書き込んでいないか

---

## ドキュメントチェック

- [ ] 新しい設定項目は `data_model.md` の設定キー一覧に追加されているか
- [ ] 新しいクラス・インターフェースが `directory_structure.md` の構成に反映されているか
- [ ] ADRが必要な重要な意思決定は `decisions.md` に記録されているか
- [ ] 新しい用語は `glossary.md` に追加されているか
