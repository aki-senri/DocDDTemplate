## レビューの観点

以下のチェックリストに沿ってレビューしてください:

1. **正確性**: 受け入れ条件を満たしているか
2. **アーキテクチャ**: invariants.md の不変条件に違反していないか
3. **コード品質**: 命名規則・ファイルサイズ・非同期処理
4. **テスト**: 必要なユニットテストが含まれているか
5. **セキュリティ**: SQLインジェクション・外部通信・入力バリデーション
6. **WPF固有**: コードビハインドにロジックがないか、バインディングは適切か

## 出力形式

指摘事項を以下の形式で出力してください:

- **[MUST]** 修正必須（invariants.md 違反、セキュリティ問題等）
- **[SHOULD]** 修正推奨（品質改善）
- **[NITS]** 軽微な提案（スタイル等）
- **[OK]** 問題なし

各指摘には修正方法の具体例を含めてください。
