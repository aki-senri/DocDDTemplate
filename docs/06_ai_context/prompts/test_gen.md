## 生成してほしいテストケース

1. 正常系（基本パス）
2. 境界値（空文字・null・最大長・最小値・最大値）
3. 異常系（例外が期待されるケース）
4. 状態変化の確認（モックの呼び出し検証を含む）

## テスト実装の制約

- xUnit の `[Fact]` / `[Theory]` / `[InlineData]` を適切に使用すること
- `Moq` でインターフェースをモックすること
- テストメソッドの命名は `{メソッド}_{シナリオ}_{結果}` に従うこと
- 各テストは独立して実行可能であること（テスト間の依存禁止）
- 非同期メソッドのテストは `async Task` で書くこと
